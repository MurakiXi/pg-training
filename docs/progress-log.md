- Week1で学んだDOM操作・イベント処理・関数・条件分岐を、別題材でも再現できる。
- 共通 class と `querySelectorAll` を使うと、ボタンの順序や個数に依存しにくい処理になる。
- `event.target.textContent.trim()` と `correctAnswer` の比較により、クリックされた選択肢の文字で正誤判定できる。
- `showResult` のような表示用関数を作ると、結果表示と解説表示をまとめて扱える。
- 動作確認では、通常のクリックだけでなく、ボタン順序変更や空白・改行の影響も確認するとよい。

### 詰まった点・注意点

- `script2.js` を使う場合は、今後の管理上その理由を明確にする。基本的には `script.js` に統一した方が分かりやすい。
- `showResult("正解！", reasonText);` のように、文末セミコロンを揃える。
- 子供向け解説文は、正確さだけでなく長さと読みやすさも考える必要がある。
