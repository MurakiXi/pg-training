## 2026-06-04

### 完了したこと

- Week1 Day2 として、結果と理由を表示する処理を `showResult(msg)` 関数にまとめた。
- `showResult(msg)` の中で、`result.textContent` に引数 `msg` を表示し、`reason.textContent` に共通の `reasonText` を表示するようにした。
- 3つの `addEventListener` の中から `showResult("正解！")` または `showResult("残念！")` を呼び出す形に修正した。
- `addEventListener("click", showResult("正解！"))` と書くと、クリック前に `showResult` が実行されてしまうことを確認した。
- `function () { showResult("正解！"); }` の形にすることで、クリック時に処理を実行できるようにした。
- 読み込み直後は結果・理由が表示されず、各ボタンをクリックした時だけ正誤判定と解説が表示されることを確認した。

### 学んだこと

- 関数呼び出し `showResult("正解！")` は、その場で即実行される。
- `addEventListener` の第2引数には、「今すぐ実行した結果」ではなく、「後で実行する関数」を渡す必要がある。
- 引数を使うと、共通処理の中に違う値だけを渡せる。
- 重複していた表示処理を関数にまとめることで、クリック処理の見通しが良くなる。

### 詰まった点・注意点

- `addEventListener("click", showResult("正解！"))` と書くと、クリックを待たずに関数が実行された。
- エラーが出ない場合でも、処理の実行タイミングが間違っていることがある。
- 「関数を呼び出す」と「関数を渡す」は別物として区別する必要がある。
